快速幂1

2020-03-02

  • 前置知识
    一个十进制数可以转化成一个二进制数 如9 →\rightarrow→ 1001

考虑一个问题
xnx^nxn其中 0≤x,n<2310≤x,n<2^{31}0≤x,n<231 答案对1e9+7取模

显然 直接乘的复杂度为Θ(n)\Theta (n)Θ(n)的 不够优
我们把n转化成二进制
如16 →\rightarrow→ 10000
可以发现x16x^{16}x16 = x8x^8x8* x8x^8x8 = x4x^4x4 * x4x^4x4 * x4x^4x4 * x4x^4x4=....
发现有大量的需要重复计算的地方
考虑优化
从末尾开始,对于n的二进制下的每一个1,将x2len−1x^{2^{len-1}}x2len−1乘入答案,后将该位置的1变为0(其中len为从右往左数第一个1的位置)
直到全为0为止

当然 在实现上是有点区别的

#define ll long long
const int mod =1e9+7;
ll qpow(int x,int n)
{
       ll ans =1;
       ll t =x;//累乘器
       while(n)
       {
           if(n&1)//该位有1
           ans = ans * t % mod  ;
           t =t * t % mod ;
           n = n / 2 ;
       }
       return ans ;
}

实际上，还有另一种做法:
由于题目有对答案取模
我们打表发现
当n足够大时
尾数会呈现周期性变化
将周期省略，减少计算次数,复杂度Θ(不知道)\Theta (不知道)Θ(不知道)
并不推荐这种写法，这种写法需要一些空间

#define ll long long 
const int maxn=1e5+7;//看情况进行选择
const int mod =1e9+7;
int a[maxn];
ll qpow(int x,int n)
{
    memset(a,0,sizeof(a));
    a[1]=x%mod;
    a[2]=x%mod*x%mod;
    if(a[1]==a[2])return a[1];
    int cnt=2;
    while(1)
    {
        cnt++;
        a[cnt]=a[cnt-1]*x%mod;
        if(a[cnt]==a[1]){cnt--;break;}
    }
    n=n%cnt;
    return a[n];
}
