区间dp

2020-03-11 4 min read

首先可以明确的一点是:对于一段区间,开始位置,最后位置.区间长度,这三个元素知二得一

我们来看一道例题;

[USACO16OPEN]248 G

题意
给定一个1*n的地图，在里面玩2048，每次可以合并相邻两个（数值范围1-40），问最大能合出多少。注意合并后的数值并非加倍而是+1，例如2与2合并后的数值为3。
输入
共n+1行
第一行为n
剩下n行为地图中的元素
输出
合出的最大值
样例:
输入

4
1
1
1
2

输出

3

区间dp，实际上是将一种合区间的操作(自动联想并查集，并查集是合并集合)
对于初始状态 每一个元素都是一个区间
在特定条件下(数字相同时)合并来求得答案
考虑如何表征一个区间
不妨设f[i][j]为第i个位置到第j个位置的元素所构成区间
由于一个区间是通过两个区间合并构成的
所以f[i][j]是由两个区间合并构成的
但两个区间的值不同 合并出来的值也会不同
所以f[i][j]会有多值
但f[i][j]作为下次合并的元素时,元素需要越大越好
所以根据题意我们只需要求得f[i][j]的最大值
我们直接设f[i][j]为i->j的最大值
容易得到状态转移方程

if(f[i][k]==f[k+1][j])
f[i][j]=max(f[i][j],f[i][k]+1)

其中i<=k<=j
现在设计好了状态 方程也得出了 考虑实现
一种比较智障的方式(我没学习前)是
i,j,k分别循环然后一股脑塞入方程
大概是这样

for(int =i=1;i<=n;++i)
{
    for(int j=1;j<=i;++j)
    {
        for(int k=i;k<=j;++k)
        {
         if(f[i][k]==f[k+1][j])
        f[i][j]=max(f[i][j],f[i][k]+1)
        ans =max(f[i][j],ans);
        }
    }
}
print(ans);

非常容易发现这样的实现方式是错误的
我们不妨举个例子:
我现在需要更新f[1][3]的区间最大值
那么我需要的是f[1][1],f[1][2],f[2][3]
但f[2][3]并没有值
所以这样的做法是错误的
让我们回头看看方程

if(f[i][k]==f[k+1][j])
f[i][j]=max(f[i][j],f[i][k]+1)

上述已经说了
是将小区间合并成大区间
描绘区间的大小是用"长度"来刻画的
不妨从长度开始枚举

for(int i=1;i<=n;++i)//初始化
f[i][i]=a[i];
for(int len=2;len<=n;++i)//长度
{
    for(int l=1;l+len-1<=n;++l)//左边
    {
        int r =l+len-1;
        for(int k=l;k<r;++k)//中间节点 设f[l][k]为左区间 那么f[l+1][r]为右区间
        if(f[l][k]==f[k+1][r])
        {
            f[l][r]=max(f[l][r],f[l][k]);
            ans =max(ans,f[l][r])
        }
    }
}
